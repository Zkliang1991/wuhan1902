# 项目名称：网易云音乐

## 技术
```
1.前后端分离独立完成项目,后期维护更方便

2.使用了vue-cli脚手架进行开发

3.登陆注册加密token传入后台,方便判断用户身份信息

4.请求session判断,避免造成信息外泄

5.登陆sessionStorage判断，人性化跳转

6.使用组件开发，使用Vuex对状态进行集中式管理

7.利用axios进行数据请求,同时也对sessiontoken进行了判断也对请求进行了拦截

8.利用了vant-ui,页面整体视觉效果较好

9.写了几个请求网易云数据的接口，可以接收到音乐的各类信息

10.在github上自学了aplayer播放器并植入项目当中

11.可以通过修改请求参数搜索到更多的音乐

12.项目拥有排行榜，推荐歌单，这些都是通过网易云接口接到的数据，同时也会随着网易云的数据更新进行同步更新

13.只要是无版权问题的音乐都可以搜索出来播放和收藏

14.用户可以自定义上传头像

15.swiper渐变轮播

16.vant-ui轮播引导页
```
## 难点
```
1.对于vux中的Getters运用还不是很熟练

2.测试接口的时候数据会有异常

3.代码的处理一定要谨慎

4.vux还没有完全理解透彻 还是用到了data 因为页面刷新会导致vuex的数据流失

5.在代码的优化上面为了逻辑清晰没有用到简洁语法

6.前后端交互跨域问题

7.async的使用

```
## 心得体会
```
在写一个项目的时候一定不能着急下手,一定要想清楚思路和需要用到的模块，
逻辑思维一定要清晰，能重用的代码一定要优化，每一个组件每一个路由思路
一定要缜密，每一步其实都可以log下数据，确保数据可以得到和使用。
在处理token的时候逻辑一定要思考清楚再做每一步的判断！
```
